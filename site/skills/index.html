Skill registry

何时装哪一个 skill

不要一次性把所有内容塞进 prompt。先判断现场问题，再只调用最少必要的 skill。

现场信号 推荐 skill 作用
目标很多，口径冲突 single-core-command 收束成唯一主任务
依赖可能失效 self-reliance-stack 建立替代方案与降级路径
时间极紧 crisis-mode-design 用坏条件反推最短打法
需求很大，但没有下一步 instruction-decomposition 把讨论改写成命令链
项目越做越偏 high-frequency-review 建立短周期检阅点
反馈过多，方向摇摆 closed-loop-feedback 过滤噪音，只接收有效反馈
资源争抢严重 resource-lockdown 把资源锁给主线任务
从零启动 minimum-survival-stack 先跑通最小交付链
需要先向外界证明进展 symbolic-breakthrough 优先打出可见战果
