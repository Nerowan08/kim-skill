Guide

怎么装、怎么用、什么时候切进强执行模式

最重要的不是“全量加载”，而是先判断当前任务需不需要进入 command mode，再按现场信号只装最少必要的 skill。

推荐装载顺序

  1. 会话开始先装 skills/arming-command/SKILL.md
  2. 复杂任务再补 skills/command-core/SKILL.md
  3. 根据现场信号，最多再选 1 到 2 个场景 skill
  4. 需要人工触发时再调用 commands/*.md
  5. 需要子代理时再使用 agents/*.md

5 项快速判断

进入 command mode 的信号

  • 存在紧迫交付截止
  • 有一个结果明显高于其他结果
  • 外部依赖可能掉线或迟到
  • 反馈已多到足以扰乱方向
  • 当前更需要落地推进，而不是继续发散

退出 command mode 的信号

  • 任务已不再紧迫
  • 依赖稳定且资源充足
  • 反馈已经统一
  • 当前阶段更适合探索和发散

安装

git clone https://github.com/YOUR_GITHUB_USERNAME/kim-command-skill
cd kim-command-skill
bash tests/validate.sh

Windows：

powershell -NoLogo -NoProfile -ExecutionPolicy Bypass -File tests/validate.ps1

Pages 站点发布

仓库已经内置了 .github/workflows/pages.yml 和 site/。推到 GitHub 之后，在仓库设置里把 Pages 的构建源切到 GitHub Actions 即可。
