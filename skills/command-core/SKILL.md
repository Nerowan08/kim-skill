# Command Core Skill

## 目的
当任务具备以下任一特征时，启用本 skill：

- 截止时间很近
- 任务失败成本高
- 多任务竞争同一批资源
- 外部依赖不稳定
- 需求嘈杂、版本混乱、反馈过载
- 需要尽快打出一个可见成果

本 skill 的目标不是“想得更多”，而是“更稳、更快、更集中地把事做完”。

## 总原则：任务绝对化

每次开始前，先明确：

1. 当前唯一主任务是什么？
2. 什么结果算完成？
3. 哪些工作虽然看起来重要，但现在不能抢占第一优先级？

## 默认行为

### 1. 单核指挥
- 只保留一个主目标
- 只保留一个当前版本的判断
- 避免同时推进多个“都很重要”的方向

### 2. 自力更生
- 默认外部依赖会失效或延迟
- 先寻找本地替代方案、降级路径、手工兜底方式

### 3. 指令级拆解
输出动作必须明确主语、动作、标准、顺序。

### 4. 高频检阅
定期检查目标、资源、输出、偏差。

### 5. 资源锁仓
上下文、工具、预算优先给主任务。

### 6. 最小生存栈
先做能跑起来的最小版本。

### 7. 象征成果先行
优先产出一个可展示的阶段成果。

## 推荐输出
### A. 主任务
### B. 当前阻塞
### C. 立即动作
### D. 阶段战果
### E. 检阅点
